Component - Edit

Edit 기본기능

: Edit 가능한 input text field의 기본적인 기능으로, 여러가지 내부 형태, focus되는 상태 등에 관한 컨트롤을 보여주는 예이다.

Demo Description

입력창 내부 형태

: 입력창(input text field)으로 쓰이는 여러가지 유형과 내부형태에 관해 조작하는 예를 보여준다.

  • 입력창 type : 4가지의 type을 선택하여 각 다른 목적과 형태로 쓰이는 입력창의 유형을 선택하여 확인할 수 있다.
  • 입력창 가이드 텍스트 : 사용자가 입력창에 포커스했을 때 가이드 텍스트가 사라지도록 하며, 가이드텍스트를 변경할 수 있다.
  • 입력창 내부 스타일 : 텍스트색, 배경색, 테두리색, 테두리타입과 관련한 스타일을 다르게 지정할 수 있다.

입력창 Type

		#57 - #66 
		#120 - #123 
		

		:  input 태그의 타입으로 총 30개(2013년, html5 tag포함)가 있다. 

		그 중 주로 text  edit 형태의 입력창으로 쓰이는 타입으로 text, password, hidden 그리고 textarea 태그가 있다. 

		hidden의 타입으로 설정하면, 입력창이 나오지 않으나 실제 value를 가지고 동작할 수 있다. 
		

		
		
Focus in/out

		#68 - #78 
		

		: 입력창에 커서를 두고 클릭하여 텍스트로 편집가능한 상태를 focus된 상태라고 볼 수 있다. 

		
    
			
  • 
				focusin
				focus 되었을 때(focus in) guide text가 없어지도록 하는 등의 동작을 구현하기 위해서는 foucusin()의 함수를 사용할 수 있다.
			
    • 
			
  • 
				focusout
				또한 focus out되었을 때 guide text가 다시 생성되도록 하는 등의 동작을 구현하기 위해서는 focusout()의 함수를 사용할 수 있다.
			
    • 
			
  • 	
				focusin 선택자
				focus된 상태인 요소들을 선택하기 위해서 간단하게 $(':focus') 또는 $('*:focus') 와 같은 방식을 사용할 수 있다.
  			
    • 
  		

	
		
가이드 텍스트

		#98 - #100 
		

		: input의 value에 값을 주어 입력 focus  전의 텍스트를 설정할 수 있다.
		초기화 후에 다른 텍스트로 바꿔야 하는 경우, 태그 내 value 속성값을 바꾸어주거나 text 메소드로 재설정하여 지정해줄 수도 있다.

		본 예제에서는 attr() 메소드로 input 태그 내 "value" attribute 을 바꾸는 방식으로 구현하였다.		
		

		
		
입력창 스타일 변경

		#80 - #83 
		

		: 입력창의 텍스트색, 배경색, 테두리색, 테두리 스타일 과 같은 경우, 모두  .css() 의 함수를 사용하여 재설정할 수 있다.
