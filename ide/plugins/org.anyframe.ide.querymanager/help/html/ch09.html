9.Explorering SQL files

Query Explorer View는 프로젝트 별로 설정된 Query 맵핑 XML의 Outline 및 Query ID의 사용여부, 중복 여부를 설정된 아이콘을 통해 쉽게 파악하는 기능을 제공한다.

  • Query Explorer View 열기

  • Query Explorer View 구성

  • Query Explore View Action

9.1.Query Explorer View 열기

Query Explorer View는 Query Manager Perspective를 열면 자동으로 뜨며, 다음과 같은 방법으로도 열 수 있다.

  • Eclipse 메뉴를 통해서 열기

    "Window > Show View > Other..."에서 Query Explorer 선택

9.2.Query Explorer View 구성

Query Explorer View는 다음과 같은 요소들로 구성되어 있다.

  • Query ID Tree

    Query Manager Nature를 설정한 프로젝트 내의 Query 맵핑 XML 파일들 중, Property에 등록한 파일들에 대한 정보가 트리 형태로 나타난다.

    각각의 Query ID는 DAO 클래스에서 사용여부를 알 수 있으며, 중복된 Quer ID 역시 나타낸다.

    [표제목-Query ID Icon]
    IconDescription
    		사용중인 Query ID
    		사용중이지 않은 Query ID
    		사용중이며 중복된 Query ID
    		사용중이지 않으며 중복된 Query ID

  • Query Statement

    상단의 Query ID Tree에서 ID를 선택하게 되면, 해당 ID에 대한 쿼리문이 나타나게 된다.

  • Link with Editor

    열려있는 Query XML File의 위치가 Query ID Tree에 나타난다.

9.3.Query Explore View Action

Query Explorer View는 Query XML File 내 Query에 관해 생성, 수정, 삭제 할 수 있는 Action이 있으며 이는 Query Test Editor 를 통해 생성,수정을 수행한다 .

  • Query 생성

    신규 생성을 위한 Query XML File 선택 후 마우스 우 클릭, Context Menu 에서 "Add Query" 선택 후 열려진 Query Test Editor에서 관련 정보 입력

    Query 수정

    수정을 위한 query의 queryID를 선택 후 마우스 우 클릭, Context Menu 에서 "Edit Query" 선택 후 열려진 Query Test Editor에서 관련 정보 수정

    Query 삭제

    삭제를 위한 query의 queryID를 선택 후 마우스 우 클릭, Context Menu 에서 "Remove Query" 선택

또한 Query Manager Editor 내 Design 탭을 통해 유사한 기능이 가능하다. 이는 Design 탭에서 Query XML File의 구조적인 형태를 그리드를 통하여 표시해 주며, Context Menu를 통해 Query의 추가/변경/삭제, Query 검색 및 특정 어트리뷰트 등을 추가할 수 있다.
