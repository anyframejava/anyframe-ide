11.CTIP Integration

Code Generator 에서 CTIP 서버(Hudson,Jenkins)를 연결하여 CTIP 서버에 job을 등록하고 job을 실행하기 위해서 사용한다. CTIP 서버와 통신을 하기 위해서는 anyframe-ide-ctip-integration-x.x.x.jar 파일을 이용해 설정을 해야 한다. 설치 및 설정과 관련된 자세한 사항은 Continuous Build Server의 [참고]Jenkins 설치과 [참고]CTIP 서버 연계 설정을 참조한다.

  1. Window >> Show View >> other... >> Anyframe >> CTIP Integration 을 선택한다.

  2. CTIP Integration View 가 오픈되면 다음과 같은 항목이 보인다.(정상적으로 CTIP 서버에 접속되었을 경우에 한하며 정보가 보여 지지 않을 경우 CTIP 서버 정보 및 상태를 확인 하여야 한다.)

    • job name: ctip job name

    • Build Type: build 또는 report 타입

    • Custom Workspace : workspace 경로

    • SCM Server Type : 형상서버 종류

    • SCM Server URL : SCM 서버 url

    • Poll SCM schedule : job 실행 스케쥴 정의

    • Build other project : 해당 job 수행 후 실행한 job의 이름

  3. CTIP Integration View 가 재공하는 Tool Bar Menu는 다음과 같다.

    • CTIP Server Configuration : 팝업창이 오픈되며 CTIP Server의 별칭 및 URL을 정의

    • Refresh : CTIP Server로 부터의 정보를 변경 하고자 할 때 사용

  4. CTIP Integration View 가 재공하는 Pull Down Menu는 다음과 같다.

    • Select a project : 기능을 수행하고 하는 Project를 변경 할 때 사용한다.

  5. CTIP Integration View 내 Table 에서의 Action은 다음과 같다.

    • [CTIP Server 별칭] - [URL 선택]

      • New Job: 별도의 Popup 창이 오픈되며 다음의 Build Job 정보를 입력한다.

        • Build Type* : build는 빌드를 수행하고 배포파일 생성, report는 build 기능에 JUnit test, Emma, Jdepend, PMD등의 reporting 기능도 수행

          [표제목-Build Type]
          Typebuildreport
          Ant(target) deploy all
          Maven(goal) package clean site package

        • Project Name* : ctip job name

        • Custom Workspace : workspace 경로지정

        • SCM Server Type : subversion, cvs, none 세가지 type 중 선택

        • SCM Server URL : SCM 서버 url

        • Poll SCM schedule : job 실행 스케쥴 정의

        • Build other project : 해당 job 수행 후 실행한 job의 이름 지정

      • CTIP Configure...: 별도의 Popup 창이 오픈되며 다음의 CTIP Server 설정을 수행한다.

        • MAVEN_HOME: 프로젝트에서 Maven 빌드도구를 사용하는 경우 보여지며, Maven이 설치된 경로로 설정

        • ANT_HOME: 프로젝트에서 Ant 빌드도구를 사용하는 경우 보여지며, Ant가 설치된 경로로 설정

        • CTIP URL in Email: CTIP Email Notification 시 사용할 url

      • Job Name 선택

        • Edit Job: 선택한 job 편집

        • Remove Job: 선택한 job 삭제

        • Run Job: 선택한 job을 실행한다. 빌드가 수행되면 Status는 "building"로 보여진다. 빌드 Status에 대해 scheduling이 걸려있지 않기 때문에, 빌드 결과는 refresh 버튼을 클릭하여 확인한다. 빌드가 다 수행이 되면 refresh 버튼을 눌렀을때 빌드 결과에 대한 이미지가 보여진다. 성공은 파란색, 실패는 빨간색. 해당 Plugin과 샘플이 함께 설치되며, 해당 Plugin 적용된 샘플은 프로젝트를 실행시켜봄으로써 확인할 수 있다.

참고

위 화면을 통해서 test case 가 없는 프로젝트에 대해서 job을 생성한 후 빌드했을 때, FAILURE가 발생한다. 이는 에러는 아니지만 test case에 대해 recording을 하기 위해, Junit, Emma Plugin이 출력하는 메세지이므로 test case를 사용하지 않을 경우 해당 Plugin을 사용하지 않도록 설정해주어야 한다. "[프로젝트명]_report" job의 configure에서 "Publish JUnit test result report", "Record Emma coverage report" action을 uncheck 한다.

Job 빌드 시 메모리 부족으로 Fail 한 경우

등록한 job에 대해 빌드 수행시 다음과 같은 에러가 발생 할 경우 환경변수에 MAVEN_OPTS를 -Xmx1024m으로 추가 설정해준다.

  • 환경변수 설정 

    변수이름 : MAVEN_OPTS
변수 값 : -Xmx1024m

  • 콘솔에 출력된 에러로그 

    Error occurred during initialization of VM
Could not reserve enough space for object heap
Could not create the Java virtual machine
...
