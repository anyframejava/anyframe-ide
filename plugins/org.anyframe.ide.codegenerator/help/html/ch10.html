10.Anyframe Plugin Installation

Code Generator는 Anyframe Plugin을 Install/Uninstall 하기 위해서 Plugin Installation View를 제공한다. Plugin을 설치하면 해당 Plugin과 연관된 라이브러리가 설치되며, 일부 Plugin의 경우 샘플코드를 제공하고 있다. 제공되는 샘플코드를 라이브러리와 함께 설치하고 싶은 경우, Plugin Installation View pull down menu 에서 "Install plugins with sample"에 체크한다.

  1. Window >> Show View >> other... >> Anyframe >> Plugin Installation 을 선택한다. (Package Explorer 또는 Project Explorer 에서 선택된 Project 기준으로 정보가 보여지며, 선택된 Project 가 없을 경우 맨위의 Project 기준으로 정보가 보여진다. 또한 View 상단에 기준되는 Project 명이 표시된다.)

  2. Plugin Installation View 가 오픈되면 다음과 같은 항목이 보인다.

    • 체크박스: Install/Uninstall 할 대상을 체크하는 부분으로 다수의 Plugin을 선택할 수 있다.

    • Plugin Name: Plugin 명

    • Group Id: Plugin Library의 Group Id

    • Artifact Id: Plugin Library의 Artifact Id

    • Latest: Plugin의 최신 Version

    • Installed: 설치된 Plugin Version

    • Description: Plugin의 상세 정보

  3. Plugin Installation View 가 재공하는 Tool Bar Menu는 다음과 같다.

    • Install : 선택한 Plugin들을 Project에 설치한다.

    • Uninstall : 선택한 Plugin들을 Project에서 제거한다.

    • Refresh : Plugin List 및 Pull Down Menu의 Project List를 갱신하고자 할 때 사용한다.

  4. Plugin Installation View가 제공하는 Pull Down Menu는 다음과 같다.

    • Select a project : 기능을 수행하고 하는 Project를 변경 할 때 사용한다.

    • Update Catalog List : Remote Anyframe Repository로 부터 Anyframe Plugin 목록 파일을 Update 받는 기능이 수행된다. 디폴트로 Update된 파일(plugin-catalog-x.x.x.xml)은 [사용자 계정 홈 디렉토리]/.anyframe 폴더 하위에 존재하게 된다.

    • Install plugins with sample: 샘플코드를 라이브러리와 함께 설치하고 싶은 경우 체크박스에 체크하며, Service 타입 프로젝트에서는 비활성화 된다.

  5. 설치 또는 제거할 Plugin 항목을 체크한 후, Install 혹은 Uninstall 메뉴를 클릭한다. 만약 WAS에 배포하여 사용중이라면, WAS는 stop 시킨 상태에서 수행하도록 한다. 그렇지 않은경우, 동작은 정상적으로 수행되나, 아래 6번 내용과 같은 에러가 발생하게 된다.

    • Install : Web 타입 프로젝트에서 해당 Plugin과 샘플이 함께 설치된 경우, 적용된 Plugin 샘플은 프로젝트를 실행시켜봄으로써 확인할 수 있다.

      • metadata 정보확인: 프로젝트 폴더 >> META-INF 폴더>> plugin-installed.xml 파일에 설치한 Plugin에 대한 정보가 추가된다. 

        <plugins>
  <plugin name="core">
    <groupId>org.anyframe.plugin</groupId>
    <artifactId>anyframe-core-pi</artifactId>
    <version>x.x.x</version>
    <essential>true</essential>
    <checked>false</checked>
  </plugin>
</plugins>

      • Ant 프로젝트

        • Web 타입: 프로젝트 Web App Libraries에 라이브러리 추가.

        • Service 타입: [프로젝트 Home]/lib에 라이브러리 추가되고, .classpath에 클래스패스가 추가됨.

      • Maven 프로젝트 : 설치한 Plugin과 관련된 라이브러리 정보가 pom.xml 의 <dependency>에 추가됨.

    • Uninstall : Installed 항목에서 X표시가 되며, 다음과 같은 방법으로 확인 할 수 있다.

      • metadata 정보확인: 프로젝트 폴더 >> META-INF 폴더>> plugin-installed.xml 파일에서 해당 Plugin에 대한 정보가 제거된다.

      • Ant 프로젝트

        • Web 타입: 프로젝트 Web App Libraries에 라이브러리 삭제.

        • Service 타입: [프로젝트 Home]/lib에 라이브러리 삭제되고, .classpath에 클래스패스가 삭제됨.

      • Maven 프로젝트 : 설치한 Plugin과 관련된 라이브러리 정보가 pom.xml 의 <dependency>에 삭제됨.

      • 프로젝트 하위로 uninstalled 폴더가 생겨나고, 그 하위로 "yyyyMMddHHmmss" 패턴에 해당하는 시간 별 폴더가 생겨나고 이 폴더 내에 uninstall 대상이 되는 파일들이 백업용으로 남게 된다.
