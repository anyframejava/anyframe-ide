7.Configuration

Query Manager를 올바르게 사용하기 위해서는 Query Manager Nature 부여와 Database 정보 및 Query XML File에 관한 정보가 필요하다.

  • Query Manager Nature

  • Property 설정

7.1.Query Manager Nature

Query Manager를 사용하기 위해서는 Nature 부여가 필요하며 Project 선택 후 마우스 우 클릭 , Context Menu 에서 "Anyframe > Activate Query Manager" 를 선택한다.

정상적으로 Query Manager Nature 가 부여 되면 Project명 오른쪽 하단에 "Q" 가 표시된다.

7.2.Property 설정

7.2.1.Databases

[Project Properties] > Anyframe > Databases 에서 속성 정의를 하며 Database connection profiles을 신규 등록,편집,삭제를 수행한다. Code Generator를 통해 프로젝트를 신규 생성한 경우는 기본값으로 신규 생성 시 입력된 값이 설정된다.

Database Connection Profile은 SQL문 테스트 시 활용된다.

7.2.2.Mapping XML 설정

[Project Properties] > Anyframe > Query Manager 에서 속성 정의를 하며 Query ID를 관리하고자 하는 Query 맵핑 파일을 추가할 수 있다. 여기에 추가한 XML 파일내에 있는 ID들만이 앞으로 나올 Query Explorer View에서 나타난다.

또한 Query ID의 사용 여부, 중복 여부를 보여주는 설정이 가능하며 Query ID를 새로 생성할 때 Query에 대한 정보를 주석으로 남길 것인지에 대한 설정 역시 가능하다.

  • Enable query comment append

    새로운 Query 생성시 주석 표시 여부를 결정한다.

  • Enable Query ID duplication checking

    Query ID 중복을 체크해 준다.

  • Enable Query ID used or unused checking

    DAO에서 사용되지 않는 Query ID를 체크해 준다.

XML 파일을 추가하는 방법은, 프로젝트 내의 경로를 찾아가 추가하는 방법과, 관리 대상이 될 수 있는 XML 파일들 중에 고르는 방법 두가지가 있다.

  • Add

    Query ID가 포함된 Query 맵핑 xml 파일들을 트리 형태로 보여준다.

  • Scan

    Query ID가 포함된 Query 맵핑 xml 파일들을 선택할 수 있게 보여준다.
