Anyframe Java Core CHANGELOG
==========================
http://www.anyframejava.org

Changes in version 5.5.0.RC1 (2012-10-19)
-------------------------------------
* 주요 변경 내역
1) 공통
  - 모든 Service API 호출시 발생하는 오류에 대해 Checked Exception이 아닌 Runtime Exception으로 throw하도록 변경하여
    어플리케이션 개발시 Anyframe Java Core에서 throw한 Exception을 명시적으로 catch하지 않도록 개선  
  - Java5 특징 중의 하나인 Generics를 지원하도록 Service API 변경

2) Plugin
  - 기존 util-demo plugin을 util plugin으로 변경
  - 기존 util-system과 util-system-demo plugin을 통합한 util-system plugin 제공
  - 기존 login plugin을 i18n plugin으로 변경하고 DBMessageSource, LocaleResolver를 활용하여 국제화 처리 사례 제시  
  - logback 매뉴얼 상세화

3) Service
  - QueryService에서 SQLLoader를 injection하여 사용할 수 있도록 SQLLoader 구조를 개선하여 Query Mapping 정보가 파일이 아닌 
    다른 형태로 존재할 경우에도 쉽게 처리할 수 있도록 확장 포인트 제공
  - Mip Query, XP Query를 이용하여 정의된 Procedure를 실행하였을 경우 NullPointerException이 발생하는 문제 조치
  - Miplatform/Xplatform 기반일 경우, 조회된 대량의 데이터를 사용자가 정의한 데이터 개수만큼씩 분할하여 화면으로 전달할 수 있도록 개선
    . Firstrow 방식 응답 처리 구현
  - Util 클래스 내의 다양한 메소드에 대한 리팩토링 수행  
    . 메소드 로직의 중복 구현 제거
    . 일관된 메소드명 부여
    . 다른 이름을 가졌으나 동일한 로직을 수행하는 메소드 통합 등
  - 테이블 기반으로 Unique Id를 생성하고자 할 때, 해당 테이블의 컬럼명을 직접 수정할 수 있도록 개선
  - Maven Repository에 소스 코드 라이브러리도 배포시킴으로써 해당 Service를 사용하는 개발자에게 Debugging 편의 제공
  
4) IDE
  - Plugin의 구성 요소에 대해 정의한 plugin-build.xml의 구조를 직관적이고 명시적으로 정의하고 해당 파일 처리 로직 변경
  - Plugin 삭제시 프로젝트에서 개별적으로 추가한 리소스들이 삭제되지 않도록 특정 Plugin 설치로 인해 추가된 리소스들만 삭제
  - 다른 Logging Framework을 설치하였을 때 기 설치된 Logging Framework과 관련된 정보 제거

Changes in version 5.2.0 (2012-06-15)
-------------------------------------
* 변경 내역
1) 공통
  - Spring Framework 3.0.5 -> 3.1.1 업그레이드
    
2) Plugin
  - mybatis plugin(Anyframe Java와 MyBatis 연계) 추가 제공
  - logback plugin(Anyframe Java와 Logback 연계) 추가 제공
  - xplatform plugin(XPLATFORM의 DataSet을 Map, VO로 변환하여 의존성을 최소화하는 연계 구조) 추가 제공
  - 기존 xp-query plugin 내에 화면 샘플 추가 제공
  - Cache Abstraction 기능을 활용한 ehcache 기반의 cache plugin 추가 제공 
    . 기존 oscache 기반 cache 기능은 oscache plugin 으로 이관
  - DB 기반의 scheduling 설정 기능을 제공하는 scheduling plugin 추가 제공 
  - cxf plugin에서 참조하고 있는 Apache CXF 2.3.0 -> 2.5.2 업그레이드
    
3) Service
  - QueryService Mapping XML File Reload 시점에 컬럼 정보 및 캐싱 정보 갱신 로직 개선
  - QueryService에서 제공하는 AbstractDao 클래스를 개선한 QueryServiceDaoSupport 클래스 신규 제공
  - Scheduling Service 기능 신규 제공
    . quartz api를 이용한 Job 등록, 조회, 수정, 삭제 기능 지원
    . Job의 저장소 변경 기능 지원(file, database)
    . Job 수행 결과 조회 기능 지원 
  - Spring MVC의 @EnableWebMvc를 확장한 @EnableWebMvcAnyframe 신규 제공
  - 요청받은 View를 직관적으로 서비스 할 수 있는 SimpleMediaTypeViewResolver 클래스 신규 제공
  - Test Case를 JUnit3 형식에서 JUnit4 형식으로 변경
  - Anyframe Java, Anyframe Enterprise Java 유틸리티 클래스 통합
    . StringUtil, DateUtil, NumberUtil 의 API 통합 및 변경
      (http://dev.anyframejava.org/docs/anyframe/plugin/optional/util-demo/1.0.2/reference/htmlsingle/util-demo.html)
 
    
4) IDE
  - Anyframe IDE 설치시 함께 설치되는 m2eclipse(Eclipse Plugin) 버전을 0.10.X에서 0.12.X로 변경
  - MS SQL 기반의 DB Configuration 기능 지원 
  - CRUD 소스 코드 생성시 Sample Data를 DB에 insert할 것인지 선택할 수 있는 기능 지원
  - CRUD 소스 코드 생성 템플릿 변경
    . Mybatis 기반의 DAO Framework을 지원하는 Mybatis 템플릿 추가 제공 
    . XPlatform UI 기반의 소스 코드 생성 템플릿 추가 제공
    . Map Type의 템플릿 삭제
  - CTIP 연계시 Hudson과 함께 Jenkins 연계 추가 지원

Changes in version 4.6.1 (2010-12-03)
-------------------------------------
* 서비스 변경 내역
  - Spring의 <mvc:annotation-driven> tag를 확장한 Namespace를 제공하여 Double Submit 방지 기능을 위한 설정 방법 추가
  - 쿼리 수행으로 조회된 칼럼의 정확한 Alias명을 추출할 수 있도록 하기 위해 기존의 ResultSetMetaData.getColumnName() 부분을 ResultSetMetaData.getColumnLabel()과 같이 변경
  - 쿼리서비스 내에 포함된 SQLGenerator에서 로직 수행을 위해 필요한 값을 멤버 변수에 할당하는 방법으로 처리하고 있음. 이는 QueryService가 Singleton 방식으로 동작하므로 동시에 쿼리문 수행을 요청할 때 잘못된 수행 결과를 초래할 수 있음. 따라서, AbstractPagingSQLGenerator 내에 정의된 멤버 변수를 제거하고, SQLGenerator로부터 쿼리 인자값 및 타입을 추출하는 로직을 변경함.
  - GenericService에 페이징 처리없는 목록 조회 기능, SearchVO가 아닌 Domain 객체를 이용한 목록 조회 기능 등 추가 구현
